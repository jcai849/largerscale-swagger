This is the API

getData

Attempt to retrieve data from the server. If the data is still being processed on the server, the header will be sent immediately, with the data following after becoming available.

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/data/{data-ids}/value

HTTP Method

GET

Response Type

Data

Parameters

  • Path: data-ids Identifiers

    An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

getDataStatus

Determine data availability at a server directly, with the response code being the same as that of the equivalent GET request

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/data/{data-ids}/value

HTTP Method

HEAD

Response Type

Parameters

  • Path: data-ids Identifiers

    An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

rmData

Delete data corresponding to data-ids at a server

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/data/{data-ids}/value

HTTP Method

DELETE

Response Type

Parameters

  • Path: data-ids Identifiers

    An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

sendData

Replication of data, including initial data transfers as well as replication of computation objects. Requires informing the server of the identifier of the data.

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/data/{data-ids}/value

HTTP Method

PUT

Response Type

Parameters

  • Body: body List(Data_inner)

    The serialised data to be replicated.

  • Path: data-ids Identifiers

    An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

sendLocationServiceAddress

Informs the node that the location service is located at the address given as the request body

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/location-service

HTTP Method

POST

Response Type

Parameters

  • Body: body String(string)

    The address of the location service

sendProcedure

The server will call the procedures over the associated arguments, emerging and aligning them, with possible recovery, as part of the computation object given in the request body. The result is stored under the output-ids, with the computations being stored under the data-ids provided as a parameter.

URL

https://virtserver.swaggerhub.com/j-c/largerscale/1.0.0/data/{data-ids}/call/{output-ids}

HTTP Method

PUT

Response Type

Chunks

Parameters

  • Body: body List(Computation)

    The computations to be run

  • Path: data-ids Identifiers

    An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

  • Path: output-ids Identifiers

    An array of data identifiers to store output of a call upon. Must respectively match the data-ids given for computations.
