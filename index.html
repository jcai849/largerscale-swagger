get /data/{data-ids}/locations
Determine the locations of data (getLocation)
Attain the locations of the data referenced with the data-ids, at which the data may be retrieved.

Path parameters

data-ids (required)
Path Parameter — An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.

Responses

put /data/{data-ids}/call/{output-ids}
Run procedures (sendProcedure)
The server will call the procedures over the associated arguments, emerging and aligning them, with possible recovery, as part of the computation object given in the request body. The result is stored under the output-ids, with the computations being stored under the data-ids provided as a parameter.

Path parameters

data-ids (required)
Path Parameter — An array of data identifiers to operate upon. All data must be available in order to carry out a successful operation, otherwise no operation is carried out.
output-ids (required)
Path Parameter — An array of data identifiers to store output of a call upon. Must respectively match the data-ids given for computations.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • text/plain

Request body

body Computation (required)
Body Parameter — The computations to be run

Return type

Chunks

Example data

Content-Type: application/json
[ {
  "computation" : {
    "rel" : "data",
    "href" : "07d3280b-26be-4940-8b21-27136df707b5",
    "type" : "GET"
  }
}, {
  "computation" : {
    "rel" : "data",
    "href" : "07d3280b-26be-4940-8b21-27136df707b5",
    "type" : "GET"
  }
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

202

Accepted, with computation queued. Returning Chunk objects to reference the resulting chunks. Chunks

400

Bad Request

5XX

Unexpected Error.
